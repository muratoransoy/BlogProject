A short snapshot of the tools I use day to day. This is a living document and changes with my workflow.

> Last updated: **2026-04-29**

## Machine and environment

- **OS:** Windows 11 Pro
- **Terminal:** Windows Terminal
- **Shell:** PowerShell and Git Bash
- **Editor:** VS Code

## Languages and toolchain

- **Go:** my main language for backend services, CLI tools, and learning projects.
- **JavaScript / TypeScript:** for web work and automation when needed.
- **Python:** for small scripts, data conversion, and quick experiments.
- **SQL:** mostly PostgreSQL and relational data modeling.

## Backend and infrastructure

- **PostgreSQL:** first choice when a real data model is needed.
- **Redis:** caching, queue-like flows, and fast prototypes.
- **Docker:** repeatable local development environments.
- **Cloudflare:** Pages, Workers, DNS, cache, and analytics.

## Web and publishing

- **Hugo:** static site generator for this blog.
- **Blowfish:** base theme, customized with a stronger homepage and project CSS.
- **GitHub Actions:** build verification and link checks.
- **Wrangler:** Cloudflare deployment workflow.

## Daily principle

Before asking whether a tool is popular, I ask whether it starts fast, stays understandable, and can be debugged when something breaks.
